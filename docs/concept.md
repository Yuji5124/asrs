# ASRS コンセプト

## 一言で言うと

**「小さく作って、すぐ遊べる」ブラウザ完結型のシンプルRPG制作ツール。**

ASRS（AIDA Simple RPG Studio）は、HTML5ベースで動作するRPG制作ツールです。
小さなマップを作り、イベントを置き、すぐにテストプレイできることを最も重視します。
戦闘はクラシックRPG風のフロントビュー・コマンド式戦闘を採用します。

## 核となる制作体験 — 3ステップループ

1. **置く** — 小さいマップにタイルとイベントを置く
2. **試す** — ワンクリックでテストプレイに入る
3. **直す** — すぐエディタに戻って調整する

このループが数秒で回ることが最重要。ビルド待ち・画面遷移の多さ・設定項目の多さはすべて敵。

## 設計原則

- **Small by default** — マップは16×12マス固定から始める。大きくするのは後。
- **JSON first** — ゲームの全データは人間が読めるJSON。これが将来のAI連携の土台になる。
- **エンジンとエディタの分離** — 「ゲームを実行する部分（core/player）」と「編集する部分（editor）」を最初から分ける。
- **素材ゼロで動く** — 画像がなくても色付きタイルと記号で完結する。素材は後から差し替え可能。
- **完全オリジナル** — 参考にするのはジャンルとして一般化された構造だけ。名称・見た目・データはすべてASRS独自（`originality-policy.md` 参照）。

## 目指すもの

- 初めての人でも15分で「歩けるマップ」が作れる体験
- 短編RPG（15分程度で完結）を1本作りきれるツール
- 作ったゲームをJSON 1ファイルで共有できること
- 将来的に、AI（Claude Code / Codex）が制作を**補助**できるデータ設計

## 目指さないもの（スコープ外）

- 3D、リアルタイム戦闘、オンラインマルチプレイ
- 巨大マップ、オープンワールド
- サイドビュー戦闘（フロントビューのみ）
- プラグインシステム（少なくともPhase 5まで）
- AIによる全自動ゲーム生成（AIはあくまで補助であり、作るのはユーザー）
