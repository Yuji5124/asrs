# ASRS 開発ロードマップ（Phase 1〜5）

各Phaseは「完了条件」を満たしたら次に進む。先のPhaseの機能を先取りしない。

## Phase 1: マップエディタ + 歩けるデモ

- 16×12マスのマップエディタ（タイル: 床・壁・草・水）
- 配置物: 宝箱・NPC・イベント玉・主人公スタート地点
- 左パレット / 中央マップ / 右設定パネルの3ペインUI
- テストプレイモード: 矢印キー/WASDでグリッド移動、壁と水は通行不可
- プロジェクトのJSON書き出し・読み込み、localStorage保存
- 読み込み時のvalidate（不正JSONでクラッシュしない）

**完了条件:** 「マップを作る → 保存 → リロードして復元 → テストプレイで壁を避けて歩ける」が通しで動く。

## Phase 2: イベントシステム

- イベント玉のコマンド編集UI
- イベントコマンド: メッセージ / アイテム入手 / スイッチON・OFF / 場所移動 / 戦闘開始（詳細は `event-commands.md`）
- テストプレイ中のイベント実行（メッセージウィンドウ、決定キー送り）
- スイッチによる出現条件
- 複数マップとマップ間移動

**完了条件:** 「NPCに話しかける → メッセージ → アイテム入手 → 別マップへ移動」が作れる。

## Phase 3: フロントビュー戦闘

- 上部に敵表示（当面は色付きシルエット＋名前）、下部にコマンド＆メッセージウィンドウ
- コマンド: たたかう / スキル / どうぐ / にげる
- 敵1〜3体、味方最大4人、ターン制（詳細は `battle-spec.md`）
- 演出: 敵点滅・ダメージ数字・メッセージ
- 勝利時: 経験値・お金・アイテム入手、レベルアップ
- データベース編集UI（敵・スキル・アイテム・職業・敵グループ）

**完了条件:** イベント玉から戦闘を開始し、勝敗がゲーム進行に反映される。

## Phase 4: ゲームとしての完成度

- タイトル画面、セーブ/ロード（プレイヤー側）、ゲームオーバー
- 装備・お店・宝箱の中身設定
- ゲーム全体を1つのJSONとしてエクスポート/インポート（共有可能に）
- BGM/SEの差し込み口の用意（素材はまだなくてよい）

**完了条件:** 15分程度で完結する短編RPGが1本作れて、JSONを渡せば他人も遊べる。

## Phase 5: AI連携

- プロジェクトJSONをAI（Claude Code / Codex）に渡すためのエクスポート整形
- AI出力JSONの取り込みUI（validate → 差分プレビュー → 承認 → 反映）
- validate強化: 不整合チェック（存在しないID参照、到達不能マップ等）
- プロンプトテンプレート集（NPC会話案、敵データ案、バランス調整案など）
- 詳細は `ai-integration.md`

**完了条件:** 「AIに敵データ3体を作らせて、レビューして取り込む」が安全に回る。
