# ASRS 戦闘仕様（Phase 3）

## 形式

**フロントビュー・コマンド式のターン制戦闘。**
敵が画面上部に正面向きで表示され、下部にコマンドウィンドウとメッセージウィンドウが並ぶ、クラシックRPG風の戦闘形式。
UIデザイン・敵の見た目・スキル名などはすべてASRS独自とする（`originality-policy.md` 参照）。

## 画面レイアウト

```
+-----------------------------------------------+
|                                               |
|        [敵A]      [敵B]      [敵C]            |  ← 敵を正面表示（1〜3体）
|                                               |
+-----------------------------------------------+
| 味方ステータス（最大4人: 名前 / HP / MP）        |
+---------------------+-------------------------+
| コマンドウィンドウ    | メッセージウィンドウ       |
| たたかう / スキル /  |  戦闘ログ・結果表示        |
| どうぐ / にげる      |                          |
+---------------------+-------------------------+
```

- 敵の見た目は当面、色付きシルエット＋名前ラベル（画像素材なしで動く）。
- ウィンドウの枠・配色・カーソルはASRS独自デザイン。

## コマンド

| コマンド | 動作 |
|---|---|
| たたかう | 対象の敵1体に通常攻撃 |
| スキル | MPを消費してスキル使用（対象はスキル定義による） |
| どうぐ | インベントリのアイテムを使用 |
| にげる | 確率で戦闘離脱。失敗するとそのターン行動不可 |

## 戦闘フロー

1. 戦闘開始（`startBattle` コマンドから `troopId` を受け取る）
2. ターン開始: 味方全員のコマンドを入力
3. 行動順を `speed` 順に決定し、順番に実行
4. 全行動終了後、勝敗判定 → 続くなら次ターンへ
5. 勝利: 経験値・お金・ドロップアイテムを入手。経験値が閾値を超えたらレベルアップ
6. 敗北: ゲームオーバー処理（Phase 3で確定。Phase 4でタイトルへ戻る等を整備）
7. 逃走成功: 戦闘前の状態でマップに戻る

## ダメージ計算（初期案）

- 通常攻撃: `damage = max(1, attack - defense / 2)` に ±10% の乱数
- スキル: スキル定義の `effect`（威力パラメータ）を基に同様の式で計算
- 自由な数式定義はしない（固定式＋パラメータ）。式の調整は `decisions.md` に記録する

## 演出

- 攻撃ヒット時: 対象の敵が点滅
- ダメージ数字を対象の上にポップ表示
- 行動内容と結果をメッセージウィンドウに逐次表示
- 演出は短く、テンポを優先する（スキップ可能にするかは実装時に決める）

## 決めるべきこと（未決）

- 敵AIの行動パターン記述形式（単純なローテーション / 重み付きランダム）
- 状態異常を入れるか（Phase 3では見送り推奨）
