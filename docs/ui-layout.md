# ASRS UIレイアウト設計

## 全体構成 — 2つのモード

アプリは常に以下のどちらかのモードにある。ボタン1つで切り替わり、切替は数秒以内。

- **エディタモード** — マップとイベントを編集する
- **テストプレイモード** — 作ったマップを実際に歩いて確認する

テストプレイ終了時はエディタの編集状態にそのまま戻る（編集内容が失われない）。

## エディタモード — 3ペインレイアウト

```
+-----------------------------------------------+
| ツールバー（モード表示 / 保存・読込 / JSON /    |
|             テストプレイ / ステータス）         |
+----------+----------------------+------------+
|          |                      |            |
| パレット  |    マップCanvas       | 設定パネル  |
| (左)     |    (中央)             | (右)       |
|          +----------------------+            |
|          | 情報バー（座標・サイズ）|            |
+----------+----------------------+------------+
```

- ツールバーには現在のモード（「マップ編集」チップ）を常時表示する
- 画面幅が狭い場合はツールバーが折り返し、左右パネルは幅を縮める（メディアクエリ）
- さらに狭い画面では、パレット・Canvas・設定パネルを縦に積み、各パネルをスクロール可能にする

### 左: パレット
- 「地形タイル」「配置物」「マップ設定」のセクションに分類し、区切り線で分ける
- 各項目はキャンバスと同じ色・記号のスウォッチ付きで表示（タイル=角形、配置物=円形）
- 通行不可タイル（かべ・みず）には「通行不可」バッジを表示
- 選択中の項目はハイライト＋左端のアクセントバーで示す
- パレット下部に「選択中のツール」を常時表示

### 中央: マップCanvas
- 初期サイズは32×24マス（Phase 1.2以降の初期値。データ上は `GameMap.width` / `height` が正で可変）、1タイル32px（初期サイズの描画領域 1024×768px）
- マップが表示領域より大きい場合はマップエリアをスクロールできる。小さい場合は中央寄せ
- クリックまたはドラッグで、パレットで選択中のタイル/配置物を置く
- 配置物をクリックすると選択状態になり、右パネルに情報表示
- グリッド線を表示。ホバー中のマスは枠でハイライトする
- ホバー中のマスには、選択中ツールで置かれる内容を半透明プレビューで表示する（既存配置物の上は選択を優先する）
- カーソル形状はツールに応じて変える（塗り=crosshair / 配置=copy / 配置物の上=pointer）
- Canvas下の情報バーに「ホバー中の座標・タイル種・通行可否・配置物名」「クリック時の操作」「マップサイズ・配置物数」を表示

### 右: 設定パネル（インスペクタ）
- 選択中の配置物の情報（種別チップ・名前・メッセージ・ID・座標）を表示・編集
- 何も選択していないときはマップ全体の情報（マップ名・サイズ・スタート地点・配置物数）と操作ヒントを表示
- Phase 2以降: イベントコマンドの編集UIがここに入る

## テストプレイモード

- 中央にマップCanvas、主人公は色付き矩形で表示。エディタと同様にマップが大きい場合はスクロールできる（プレイヤー追従カメラは将来課題）
- 矢印キーとWASDでグリッド移動（Phase 1は即時移動、補間アニメは後回し）
- 壁・水・配置物のマスは通行不可
- 「エディタに戻る」ボタンを常時表示
- Phase 2以降: 下部にメッセージウィンドウが出る

## 描画方針

- **マップ描画は Canvas**（タイルの一括描画に向く）
- **UI（パレット・パネル・ウィンドウ・メニュー）は React/DOM**（状態管理と開発効率に向く）
- 画像素材なしで動く: タイルは色 + 記号（例: 水=青系、壁=濃灰）で区別。将来、画像に差し替え可能な描画層にしておく
