# ASRS UIレイアウト設計

## 全体構成 — 2つのモード

アプリは常に以下のどちらかのモードにある。ボタン1つで切り替わり、切替は数秒以内。

- **エディタモード** — マップとイベントを編集する
- **テストプレイモード** — 作ったマップを実際に歩いて確認する

テストプレイ終了時はエディタの編集状態にそのまま戻る（編集内容が失われない）。

## エディタモード — 3ペインレイアウト

```
+----------+----------------------+------------+
|          |                      |            |
| パレット  |    マップCanvas       | 設定パネル  |
| (左)     |    (中央)             | (右)       |
|          |                      |            |
+----------+----------------------+------------+
| ツールバー（保存 / 読込 / テストプレイ開始 など）|
+-----------------------------------------------+
```

### 左: パレット
- タイル選択: 床・壁・草・水（色付き矩形＋ラベルで表示）
- 配置物選択: 宝箱・NPC・イベント玉・スタート地点
- 選択中のものがハイライトされる

### 中央: マップCanvas
- 16×12マス、1タイル32px（描画領域 512×384px）
- クリックまたはドラッグで、パレットで選択中のタイル/配置物を置く
- 配置物をクリックすると選択状態になり、右パネルに情報表示
- グリッド線を表示

### 右: 設定パネル（インスペクタ）
- 選択中の配置物の情報（名前・座標・種別）を表示・編集
- 何も選択していないときはマップ全体の情報（マップ名など）
- Phase 2以降: イベントコマンドの編集UIがここに入る

## テストプレイモード

- 中央にマップCanvas、主人公は色付き矩形で表示
- 矢印キーとWASDでグリッド移動（Phase 1は即時移動、補間アニメは後回し）
- 壁・水・配置物のマスは通行不可
- 「エディタに戻る」ボタンを常時表示
- Phase 2以降: 下部にメッセージウィンドウが出る

## 描画方針

- **マップ描画は Canvas**（タイルの一括描画に向く）
- **UI（パレット・パネル・ウィンドウ・メニュー）は React/DOM**（状態管理と開発効率に向く）
- 画像素材なしで動く: タイルは色 + 記号（例: 水=青系、壁=濃灰）で区別。将来、画像に差し替え可能な描画層にしておく
