# ASRS オリジナリティ規約

## 原則

**参考にしてよいのは「ジャンルとして一般化された構造」だけ。固有表現は一切持ち込まない。**

「小さなマップを作ってすぐ遊ぶ制作体験」「フロントビュー・コマンド式戦闘」といったジャンル慣習は参考にしてよいが、特定の既存作品・既存ツールに由来する固有要素はコピーしない。

## コピーしないもの

- **固有名詞:** 既存作品の呪文名・技名・職業名・モンスター名・地名・キャラクター名。紛らわしい変形（1文字違い等）も避ける。
- **キャラクター・モンスターの見た目:** 完全オリジナルとする。当面は色付きシルエット＋幾何学形状で表現するため自然にオリジナルになるが、将来素材を入れる際も既存デザインをトレースしない。
- **UIデザイン:** ウィンドウの枠デザイン・配色・フォント・カーソル記号・レイアウト比率は独自に決める。「下部にコマンドとメッセージ」という構造自体はジャンル慣習なので使用可。特定作品のスクリーンショットと並べて「同じに見える」状態を避ける。
- **音:** 既存作品の効果音・BGMの模倣をしない。
- **素材:** 既存作品・既存ツールの画像・音声素材を使わない。
- **テキスト:** 有名なセリフ・メッセージ文言の引用をしない。
- **ツール内の用語:** エディタの機能名も独自に付ける（例:「イベント玉」はASRS独自用語）。既存ツールの機能名をそのまま使わない。

## 公開向け表現のルール

- README・アプリ内文言・公開ドキュメント・マーケティング文では、**特定の既存ゲーム名・既存ツール名を前面に出さない**。
- 「〇〇風」と特定作品名で説明しない。代わりに次の表現を使う:
  - 「クラシックRPG風」
  - 「小さなRPG制作ツール」
  - 「フロントビュー戦闘」「コマンド式戦闘」
- 内部の設計メモで構造の参考として言及するのは可（ただし公開リポジトリでは最小限に）。

## ASRS独自の命名づくり

- スキル・職業・敵などの命名規則を先に決めてから量産する（例:「動詞＋自然現象」で統一など）。規則は決まり次第 `decisions.md` に記録する。
- 命名トーン（王道ファンタジー変形 / 和風 / SF混合など）は未決。Phase 3までに決める。

## チェック運用

- 各Phaseの完了時に「既存作品と誤認されうる要素はないか」をセルフレビュー項目に入れる。
- AI出力を取り込む際も、本規約違反の固有名詞が含まれていないかをレビュー観点に含める（`ai-integration.md` 参照）。
